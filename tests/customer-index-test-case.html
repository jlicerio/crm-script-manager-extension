Customer Index Page Test Case

Manual test coverage for the customer list page at the NexGe_NG_Customers module. This case verifies page load, search, customer creation, validation, and save behavior.

Test Case Metadata

Target Page /index.php?module=NexGe_NG_Customers&action=index
Test Type Manual UI / Regression
Priority High
Precondition User is authenticated

Objective

Confirm that the customer index page loads correctly, supports finding existing customers, and allows creation and saving of a new customer record without validation errors.

Preconditions

  • User has valid CRM credentials and access to the NexGe_NG_Customers module.
  • Browser session is active and not expired.
  • At least one existing customer is available for search verification.
  • Test data is available for a new customer record.

Test Steps

  1. Open the customer index page in the CRM.
  2. Verify the page title, module name, and customer list load without error.
  3. Use search or filtering controls to locate an existing customer record.
  4. Open the customer record and confirm the detail view loads successfully.
  5. Return to the customer index page and click the action used to create a new customer.
  6. Enter valid data into the required customer fields.
  7. Save the new customer record.
  8. Confirm the new customer appears in the module and opens correctly from the list.
  9. Repeat the create flow with required fields left blank and confirm validation is shown.

Expected Results

Page Load Customer index page renders successfully with no console or UI blocking errors.
Search / Open Existing customers can be located and opened from the list view.
Create Customer New customer form opens from the index page.
Save Valid customer data saves successfully and the record persists.
Validation Missing required fields trigger a clear validation message and prevent save.
Navigation User can return to the list view without losing session state.

Notes

The exact field names and button labels should be verified against the live CRM page before finalizing the automated version of this case. This file is a manual test case template based on the customer index module URL.

Pass / Fail Criteria

  • Pass: Customer index page loads, customer creation succeeds, validation works, and the new record is visible after save.
  • Fail: Any load error, broken navigation, failed save, missing validation, or record not appearing in the list after creation.
